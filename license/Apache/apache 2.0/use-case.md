# When Should You Use Apache 2.0?

Apache 2.0 is best when you want:

- Commercial-friendly licensing
- Patent protection
- Wide adoption
- Corporate usage

Good examples:
- Large open-source projects
- APIs and SDKs
- Cloud platforms
- Infrastructure tools
- Enterprise software

Avoid Apache 2.0 if:
- You want extreme simplicity (MIT)
- You want copyleft requirements (GPL)
