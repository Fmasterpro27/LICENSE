# When Should You Use the MIT License?

MIT is best when you want:

- Maximum adoption
- Very few restrictions
- Commercial friendliness
- Easy reuse by others

Good examples:
- Open-source libraries
- Frameworks
- APIs
- Bots and tools
- SDKs

Avoid MIT if:
- You want patent protection (consider Apache 2.0)
- You want to force open-source derivatives (consider GPL)
