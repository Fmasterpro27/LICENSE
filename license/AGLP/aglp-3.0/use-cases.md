# When Should You Use AGPL-3.0?

AGPL-3.0 is best when you want:

- Strong open-source guarantees
- Open-source SaaS and web apps
- No closed-source server-side forks

Good examples:
- Web applications
- APIs and backend services
- Federated services
- Open-source SaaS platforms

Avoid AGPL-3.0 if:
- You want proprietary SaaS
- You want easy corporate adoption
- You want permissive reuse
